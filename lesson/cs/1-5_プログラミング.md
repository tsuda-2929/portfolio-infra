
## プログラムの種類

プログラムが実行される場所で
- 「クライアントサイド」
- 「サーバーサイド」を2種類に大別されます。


- クライアントサイド
    - プログラムが実行される場所
        - ユーザーの端末（スマホ・PC）
            - PC、スマホなど
            - クライアント側の端末側のCPU・メモリ・ストレージが利用される
                - ユーザーはインターネットにつながなくても利用可能
                - ユーザー端末のスペックに依存するため軽量な処理しか動かない。
    - OSへの指示
        - ユーザー端末上で動くソフトウェア（ブラウザなど）を介して、OS（カーネル）に指示を出す
    - 主要なプログラミング言語
        - javascript
        - html
        - css

- サーバサイド
    - プログラムの実行される場所
        - サーバ上
            - EC2 ECS など
            - サーバ上のCPU・メモリ・ストレージが利用される
                - ハイスペックのサーバで動かすことで、重い処理も実行可能
                - ユーザーはインターネットにつながないと利用不可
    - OSへの指示
        - サーバ上で実行され、OS（カーネル）に直接指示を出す
    - 主要なプログラミング言語
        - javascript
        - python
        - java
        - php
        - SQL (MySQL, PostgreSQL, DynamoDB クエリなど)

- 補足    
htmlとcssはCPUを利用して、計算の命令することはできず、画面の表示形式の命令しかできない。
その点を考慮して、プログラム言語に対して、マークアップ言語と呼ばれることが通称。

## 前提

- プログラミングにおけるメモリとCPUの役割

    - メモリ
    ```
    なんらからの値を保存（値：太郎という文字列とか。数字の1とか）
    プログラムを保存（命令：ある値とある値を加算しろとか）
    ```

    - CPU
    ```
    命令を読む（メモリから）
    実行する
    次へ進む
    ※CPUはずっとこれを繰り返してる
    ※命令は「数値計算」「論理演算」「比較」「命令を飛ばす」「カウンタ（メモリに置いてある次の命令）の更新」を用いて複雑な処理を実現。
    ```

## プログラムの記述方法


> プログラミングとは、
> 
> コンピューターで行いたい目的の作業を実現するために、
> 
> 「2.データ構造」に従った「1.変数」に保存されたデータを操作する「3.基本命令」を記述することである。

- コードリーディングのTips
    - この定義に従って、下記ステップで読むことを推奨する
        - Step1.何を実現したいのか？（課題と目的） 
        - Step2.扱われているデータ構造は？（データ構造） 
        - Step3.データに対して処理されている内容は？（基本命令） 
            - いきなりStep3を理解しようと焦らない
            - システムのドメイン（業界・業務）の理解が深いと助けになったりもする
- 後述のポイント
    - 1～3が重要。
    - 4～6はあくまで1～3をより効率よく記述するために編み出された手法。
        - 事実、昔からあるシェルスクリプトは「5.クラス」を記述できない

### 1.変数

メモリに一時的に保存する場所を確保。
データ構造に従った値を格納する。

    ローカル変数：一時的にメモリに格納する変数
    環境変数：プログラムの実行される環境（OS）に設定されている変数

### 2.データ構造（値と型）

メモリに保存する値の形式を指定

    数値：数字データ形式
    文字列：文字列データ形式
    配列：複数の変数が入っているデータ形式。
    連想配列（辞書、リスト）：配列の構造に対して、変数ごとに検索するためのKeyを付与したもの。

### 3.基本命令

メモリに保存するCPUへの命令の種類。

    足し算・引き算：CPUの「数値計算」「論理演算」を組み合わせて実現
    条件分岐：CPUの「比較」「命令を飛ばす」を組み合わせて実現
    繰り返し：CPUの「比較」「命令を飛ばす」「カウンタの更新」を組み合わせて実現

### 4.関数（メソッド）

複数の基本命令または関数を再利用しやすい形にまとめたもの

    引数：関数への入力値
    返り値：実行結果を値として戻ってくる

### 5.クラス

変数と関数を名前を付けてまとめたもの

    クラス：まとめただけの記述。
    オブジェクト（インスタンス）：クラスをもとに、実際にコード上で利用できる状態にしたもの。

### 6.ライブラリ

既に誰が作ってくれた便利なクラス・関数をまとめたもの。
